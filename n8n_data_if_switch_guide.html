N8N 데이터 구조 — JSON & 아이템

N8N에서 데이터가 어떻게 생겼고 어떻게 흘러가는지 이해합니다
JSON이란?

이름표가 붙은 데이터 상자

JSON은 데이터를 담는 표준 형식이에요.
"이름": "값" 쌍으로 구성되고,
중괄호 { } 로 감쌉니다.

아이템(Item)이란?

N8N 데이터의 최소 단위

노드와 노드 사이를 흐르는 데이터 한 덩어리를 아이템이라고 해요.
아이템 하나 = JSON 객체 하나입니다.

JSON 기본 구조 
{
  "이름": "홍길동",          // 문자열 (String)
  "나이": 30,               // 숫자 (Number)
  "재직중": true,           // 참/거짓 (Boolean)
  "취미": ["독서", "등산"],  // 배열 (Array)
  "주소": {                   // 중첩 객체 (Object)
    "시": "서울",
    "구": "강남구"
  }
}
타입예시설명N8N 사용 예
String "홍길동" 큰따옴표로 감싼 문자 이름, 주소, 메시지
Number 1450.50 따옴표 없는 숫자 환율, 가격, 나이
Boolean true / false 참 또는 거짓 재직여부, 완료여부
Array ["a","b","c"] 대괄호 [ ] 목록 태그 목록, 취미 목록
Object {"시":"서울"} 중괄호 { } 중첩 주소, 설정값 묶음
💡 아이템 리스트: 고객이 10명이면 아이템이 10개 흐릅니다. 각 아이템은 한 고객의 JSON 데이터예요. 노드는 기본적으로 아이템을 하나씩 처리하고 결과를 내보냅니다.
N8N 아이템 리스트 구조 (내부 형태) 
[
  { "json": { "이름": "홍길동", "점수": 85 } },  // 아이템 1
  { "json": { "이름": "김영희", "점수": 72 } },  // 아이템 2
  { "json": { "이름": "이철수", "점수": 55 } }   // 아이템 3
]

표현식 (Expression) 완전 정복

{{ }} 이중 중괄호 안에 코드를 써서 데이터를 동적으로 참조합니다
고정값 모드

항상 같은 값

텍스트 필드에 "안녕하세요"를 직접 입력하면 항상 그 문자열만 출력됩니다. 변하지 않아요.

표현식 모드

실행 시 동적으로 계산

{{ $json.이름 }}을 입력하면 실행할 때마다 현재 아이템의 이름 필드 값을 읽어옵니다.

자주 쓰는 표현식 모음 
// ─── 현재 아이템 데이터 접근 ───
{{ $json.이름 }}              // 현재 아이템의 "이름" 필드
{{ $json.주소.시 }}           // 중첩 객체 접근
{{ $json.취미[0] }}           // 배열 첫 번째 항목
{{ $json.취미.length }}       // 배열 길이

// ─── 시간 관련 ───
{{ $now }}                    // 현재 시간 (ISO 형식)
{{ $now.format('yyyy-MM-dd') }}  // 2025-01-15
{{ $now.format('HH:mm') }}    // 09:00

// ─── 문자열 합치기 ───
{{ "안녕하세요 " + $json.이름 + " 님!" }}

// ─── 숫자 계산 ───
{{ $json.가격 * 1.1 }}       // 10% 추가
{{ Math.round($json.환율) }}  // 반올림

// ─── 특정 노드 데이터 참조 ───
{{ $('HTTP Request').item.json.title }}  // 다른 노드 데이터

// ─── 실행 정보 ───
{{ $runIndex }}              // 몇 번째 실행인지 (0부터)
{{ $itemIndex }}             // 몇 번째 아이템인지
EXPRESSION DEMO — 입력 데이터: {"이름":"홍길동","점수":85,"가격":10000}
표현식 {{ $json.이름 }}
→ "홍길동" 이름 필드 값 출력
표현식 {{ "안녕하세요 " + $json.이름 + "님, 점수는 " + $json.점수 + "점입니다." }}
→ "안녕하세요 홍길동님, 점수는 85점입니다."
표현식 {{ $json.가격 * 1.1 }}
→ 11000 10% 부가세 포함 가격
💡 표현식 모드 전환 방법: N8N 노드 설정에서 입력 필드 오른쪽의 = 아이콘 또는 fx 아이콘을 클릭하면 표현식 모드로 바뀌고 {{ }} 를 쓸 수 있게 됩니다.

IF 노드 — 조건 분기

"만약 ~이라면" 조건에 따라 데이터를 두 경로로 나눕니다
🔀 IF 노드
조건: {{ $json.점수 }} >= 60
TRUE ✓
✅ 점수가 60 이상
"합격" 처리
FALSE ✗
❌ 점수가 60 미만
"불합격" 처리
=
Equal (같다)
점수 = 100
≠
Not Equal (다르다)
상태 ≠ "완료"
>
Larger Than (크다)
가격 > 10000
<
Smaller Than (작다)
재고 < 5
⊇
Contains (포함)
이름 contains "김"
∅
Is Empty (비어있다)
이메일 is empty
AND / OR 다중 조건 
// AND 조건: 두 조건 모두 참이어야 TRUE
조건1: {{ $json.점수 }} >= 60
조건2: {{ $json.출석 }} = true
→ 점수가 60 이상 AND 출석이 참인 경우에만 합격

// OR 조건: 하나만 참이면 TRUE
조건1: {{ $json.점수 }} >= 90
조건2: {{ $json.특별상 }} = true
→ 고득점 OR 특별상 수상자면 우수상 수여
⚠️ 자주 하는 실수: 숫자를 비교할 때 값 타입을 꼭 확인하세요. "60"(문자열)과 60(숫자)은 다릅니다. API에서 가져온 숫자가 문자열로 되어 있으면 Number($json.점수)로 변환하세요.

Switch 노드 — 여러 경우 분기

IF는 2개, Switch는 N개의 경로로 분기합니다
IF 노드

참 / 거짓 — 2가지

조건 하나. 결과는 무조건 TRUE 또는 FALSE 두 갈래로만 나뉩니다.

Switch 노드

Rule 1 / 2 / 3... — N가지

여러 조건을 순서대로 체크해서 처음 맞는 경로로 보냅니다. 없으면 Fallback 처리.

Switch 노드 — 요일별 처리 예시 
// Switch 노드 설정값 (Rules 모드)

Value: {{ $json.요일 }}

Rule 1: {{ $json.요일 }} = "월요일"  → Output 0 ("한 주를 시작합니다!")
Rule 2: {{ $json.요일 }} = "금요일"  → Output 1 ("불금입니다! 🎉")
Rule 3: {{ $json.요일 }} = "토요일"  → Output 2 ("주말 시작!")
Rule 4: {{ $json.요일 }} = "일요일"  → Output 3 ("내일 월요일...")
Fallback                            → Output 4 ("평범한 평일")
Switch 노드 — 점수로 등급 나누기 
Rule 1: {{ $json.점수 }} >= 90   → Output 0 (등급: "A", "매우 우수")
Rule 2: {{ $json.점수 }} >= 80   → Output 1 (등급: "B", "우수")
Rule 3: {{ $json.점수 }} >= 70   → Output 2 (등급: "C", "양호")
Rule 4: {{ $json.점수 }} >= 60   → Output 3 (등급: "D", "미흡")
Fallback                     → Output 4 (등급: "F", "불합격")

// 주의: Switch는 위에서 아래로 순서대로 체크합니다.
// 95점은 Rule 1 (>=90)에서 먼저 걸림 → Output 0으로 이동
💡 Fallback 설정: Switch 노드에서 어떤 Rule에도 해당되지 않는 경우를 위해 반드시 Fallback 출력을 설정해두세요. 없으면 해당 아이템이 그냥 사라집니다!

HTTP Request & API 연동 기초

외부 서비스와 데이터를 주고받는 N8N의 핵심 노드
API란?

프로그램 사이의 소통 창구

API는 두 서비스가 데이터를 주고받는 규칙이에요. N8N이 날씨 서버에 "서울 날씨 알려줘"라고 요청하면, 서버가 JSON으로 응답하는 구조입니다.

HTTP Request 노드

N8N의 인터넷 연결 도구

HTTP Request 노드는 N8N에서 어떤 API든 호출할 수 있는 범용 노드입니다. URL만 알면 400개 이상의 서비스와 연동 가능해요.

GET vs POST 차이점
// ─── GET 요청: 데이터를 가져올 때 (조회) ───
Method: GET
URL: https://api.openweathermap.org/data/2.5/weather
Query Params:
  q: Seoul         // URL에 붙는 파라미터: ?q=Seoul&appid=xxx
  appid: 발급받은API키
  units: metric   // 섭씨 온도
  lang: kr        // 한국어 결과

// → 최종 요청 URL
https://api.openweathermap.org/data/2.5/weather?q=Seoul&appid=xxx&units=metric

// ─── POST 요청: 데이터를 보낼 때 (전송/생성) ───
Method: POST
URL: https://api.telegram.org/bot{TOKEN}/sendMessage
Body (JSON):
{
  "chat_id": "123456789",
  "text": "안녕하세요!"
}
// Body는 URL에 노출되지 않고 요청 본문에 숨겨 전송됨
Query Parameters

URL 뒤에 붙는 조건값

GET 요청에서 검색 조건, 필터, API 키 등을 전달할 때 사용해요.
?key=value&key2=value2 형태로 URL 뒤에 붙습니다.

Headers

요청의 메타정보 & 인증

API 인증 토큰을 전달하거나 데이터 형식을 알릴 때 사용해요.
Authorization: Bearer {토큰}이 가장 많이 쓰입니다.

Headers 설정 예시
// N8N HTTP Request 노드 → Headers 탭에 추가

// 1. Bearer Token 인증 (가장 일반적)
Authorization: Bearer eyJhbGciOiJIUzI1...

// 2. API Key 방식
X-API-Key: your-api-key-here

// 3. JSON 데이터 전송 선언
Content-Type: application/json

// 4. 노션 API 예시
Authorization: Bearer secret_xxxxx
Notion-Version: 2022-06-28
Content-Type: application/json
🌤 실습 6 — 공공 날씨 API 연동 (OpenWeatherMap)
OpenWeatherMap API로 서울 날씨를 가져오는 설정입니다.
무료 API 키 발급: openweathermap.org → 회원가입 → API Keys 탭 (발급 후 약 10분 대기)
HTTP Request 노드 설정
// ─── HTTP Request 노드 설정 ───
Method: GET
URL: https://api.openweathermap.org/data/2.5/weather

// Query Parameters 추가
q: Seoul
appid: 여기에_API_키_입력
units: metric
lang: kr

// ─── 응답 JSON 구조 ───
{
  "name": "Seoul",              // 도시명
  "main": {
    "temp": 15.3,             // 현재 기온 (섭씨)
    "humidity": 65,           // 습도 (%)
    "feels_like": 13.8        // 체감 온도
  },
  "weather": [{
    "description": "맑음"      // 날씨 설명
  }],
  "wind": { "speed": 3.5 }    // 풍속 (m/s)
}

// ─── Set 노드로 필요한 필드만 추출 ───
도시: {{ $json.name }}
기온: {{ $json.main.temp }}
날씨: {{ $json.weather[0].description }}
습도: {{ $json.main.humidity }}
💡 API 키 없이 테스트하려면: https://open.er-api.com/v6/latest/USD 이 URL은 인증 없이 환율 데이터를 바로 가져올 수 있어요!
💱 실습 7 — 환율 API로 달러→원화 변환
API 키가 필요 없는 무료 환율 API를 사용합니다. 응답에서 원하는 필드만 추출하는 방법도 함께 배웁니다.
환율 API 전체 워크플로우
// ① HTTP Request 노드
Method: GET
URL: https://open.er-api.com/v6/latest/USD
// API 키 불필요! 바로 실행 가능

// ② 응답 데이터 구조
{
  "base_code": "USD",
  "rates": {
    "KRW": 1452.5,   // 달러→원화
    "EUR": 0.924,    // 달러→유로
    "JPY": 157.3     // 달러→엔화
  }
}

// ③ 원하는 필드 추출 (Set 노드 또는 Code 노드)
// 방법 A: Set 노드 표현식
달러_원화: {{ $json.rates.KRW }}
유로_원화: {{ Math.round($json.rates.KRW / $json.rates.EUR) }}

// 방법 B: Code 노드
const rates = items[0].json.rates;
return [{
  json: {
    USD_KRW: Math.round(rates.KRW),
    EUR_KRW: Math.round(rates.KRW / rates.EUR),
    JPY_KRW: Math.round(rates.KRW / rates.JPY * 100)
  }
}];
📤 응답 데이터에서 원하는 필드 추출하기
💡 API 응답에는 필요 없는 데이터가 많습니다. Set 노드를 연결해서 필요한 필드만 골라내는 것이 좋은 습관이에요. HTTP Request 노드 OUTPUT 탭에서 실제 필드명을 확인하고 복사해서 쓰세요.
중첩 JSON에서 깊은 필드 접근
// 응답 데이터 예시
{ "weather": [{ "description": "맑음" }], "main": { "temp": 15 } }

// 배열 안의 값: [0]으로 첫 번째 요소 접근
{{ $json.weather[0].description }}  → "맑음"

// 중첩 객체: . 점으로 계속 들어가기
{{ $json.main.temp }}               → 15

// 존재 여부 확인 후 접근 (안전한 접근)
{{ $json.main?.temp ?? '데이터 없음' }}

// 숫자 포맷팅
{{ $json.rates.KRW.toFixed(2) }}     → "1452.50"
{{ Math.round($json.rates.KRW).toLocaleString() }}  → "1,453"

데이터 가공 노드: Set & Code & Loop

데이터를 추가·변경·삭제하고 대량 처리까지
📝
Set 노드 (Edit Fields) — 필드 추가 / 값 변경
새 필드 추가

없던 데이터를 만들기

기존 데이터에 없는 새로운 필드를 추가합니다.
예: API에서 온 데이터에 수집시간 필드 추가

Edit Fields (Set)

불필요한 필드 제거

"Keep Only Set Fields" 옵션을 ON 하면 설정한 필드만 남기고 나머지를 모두 삭제합니다. 깔끔한 데이터 정리에 유용해요.

Set 노드 활용 패턴
// ─── 패턴 1: 필드 추가 ───
수집시간: {{ $now.format('yyyy-MM-dd HH:mm') }}
처리상태: "완료"
환경: "production"

// ─── 패턴 2: 기존 필드 변환 ───
가격_부가세포함: {{ Math.round($json.가격 * 1.1) }}
이름_전체: {{ $json.성 + " " + $json.이름 }}

// ─── 패턴 3: 조건부 값 설정 ───
등급: {{ $json.점수 >= 90 ? "VIP" : "일반" }}
메시지: {{ $json.재고 > 0 ? "재고있음" : "품절" }}

// ─── 패턴 4: Keep Only Set Fields ON ───
// 이 필드들만 남기고 나머지 전부 삭제
이름: {{ $json.이름 }}
이메일: {{ $json.이메일 }}
// → API 응답의 수십 개 필드 중 2개만 유지
💻
Code 노드 — JavaScript로 자유롭게 처리
💡 언제 Code 노드를 쓰나요? Set 노드로 해결하기 복잡한 경우에 씁니다. 여러 필드를 동시에 계산하거나, 배열을 가공하거나, 조건 분기가 많을 때 특히 유용해요.
실습 8 — 이름+성 합치기 & 날짜 포맷 변경
// ─── Code 노드 기본 구조 ───
for (const item of items) {
  // item.json 에 현재 아이템 데이터가 있음

  // 이름 합치기
  item.json.전체이름 = item.json.성 + " " + item.json.이름;

  // 이메일에서 도메인 추출
  item.json.도메인 = item.json.이메일.split("@")[1];

  // 날짜 포맷: "2025-01-15" → "2025년 01월 15일"
  const [y, m, d] = item.json.입사일.split("-");
  item.json.입사일_한국형 = `${y}년 ${m}월 ${d}일`;

  // 가격에 부가세 추가 & 천단위 포맷
  const vat = Math.round(item.json.가격 * 1.1);
  item.json.가격_부가세 = vat.toLocaleString() + "원";
}

return items; // ← 반드시 마지막에 return items!
Code 노드 — 점수별 등급 자동 분류
for (const item of items) {
  const score = item.json.점수;

  // 조건 분류
  if (score >= 90) {
    item.json.등급 = "A";
    item.json.메시지 = "매우 우수 🏅";
  } else if (score >= 80) {
    item.json.등급 = "B";
    item.json.메시지 = "우수 👍";
  } else if (score >= 60) {
    item.json.등급 = "C";
    item.json.메시지 = "양호";
  } else {
    item.json.등급 = "F";
    item.json.메시지 = "재시험 필요 😢";
  }

  // 부가세 계산도 함께
  item.json.부가세포함가격 = Math.round(item.json.price * 1.1);
}

return items;
⚠️ 자주 하는 실수 — return items 빠뜨리기: Code 노드 마지막에 반드시 return items;를 써야 합니다. 없으면 노드 실행이 실패해요. 디버깅 시엔 console.log(item.json.값)으로 Execution 로그에서 값을 확인할 수 있습니다.

Loop 노드 (SplitInBatches) — 대량 데이터 처리
언제 쓰나요?

API 속도 제한 대응

API는 보통 분당 호출 횟수 제한(Rate Limit)이 있어요. 100개 아이템을 한번에 처리하면 오류가 납니다. 10개씩 끊어서 처리하면 안전해요.

동작 원리

배치 단위로 순차 처리

아이템 리스트를 설정한 크기로 나눠서 순서대로 처리합니다. 마지막 처리 노드를 다시 SplitInBatches에 연결하면 루프가 완성됩니다.

실습 9 — 10개 아이템 배치 처리 & Wait 노드 조합

실제 Code 노드 코드

        
for (const item of items) {
  // 이름 합치기
  item.json.전체이름 = item.json.성 + " " + item.json.이름;

  // 이메일에서 도메인 추출
  item.json.도메인 = item.json.이메일.split("@")[1];

  // 날짜 포맷: "2023-06-15" → "2023년 06월 15일"
  const [y, m, d] = item.json.입사일.split("-");
  item.json.입사일_한국형 = `${y}년 ${m}월 ${d}일`;

  // 부가세 포함 가격 계산
  const vat = Math.round(item.json.가격 * 1.1);
  item.json.가격_부가세 = vat.toLocaleString() + "원";
  item.json.부가세금액 = (vat - item.json.가격).toLocaleString() + "원";
}

return items;
